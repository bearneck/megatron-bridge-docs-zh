# GLM-4.5V

[GLM-4.5V](https://huggingface.co/zai-org/GLM-4.5V) 是一个基于 GLM-4.5 Air 架构构建的强大视觉语言模型。GLM-4.5V 结合了一个 106B 参数的稀疏混合专家（MoE）语言模型和一个视觉编码器，以实现对图像和视频的鲁棒多模态理解。

GLM-4.5V 支持多模态任务，包括图像描述、视觉问答、OCR、视频理解和通用视觉语言推理。该模型利用多分辨率旋转位置编码（MRoPE）来增强空间理解。

GLM 系列模型通过 Bridge 系统得到支持，具有自动检测的配置和权重映射。

```{important}
请将 `transformers` 版本更新至 4.57.1 或更高版本，以便使用 GLM-4.5V 模型。
```

## 可用模型

### 视觉语言模型
- **GLM-4.5V** (`zai-org/GLM-4.5V`): 106B 参数视觉语言模型（基于 GLM-4.5 Air）
  - 46 个解码器层，4096 隐藏大小
  - 96 个注意力头，8 个查询组（GQA）
  - 128 个 MoE 专家，包含共享专家
  - 每令牌约 12B 激活参数
  - 序列长度：131,072 个令牌
  - 推荐配置：32 个节点，256 个 GPU（LoRA/DoRA）或 64 个节点，512 个 GPU（全量 SFT）

## 模型架构特性

GLM-4.5V 将高效的稀疏 MoE 语言建模与多模态能力相结合：

**语言模型特性：**
- **稀疏 MoE 架构**：128 个路由专家，包含共享专家，以实现高效的参数使用
- **分组查询注意力（GQA）**：具有 8 个查询组的内存高效注意力机制
- **SiLU 门控线性单元**：使用 SiLU 激活函数的门控线性单元，以提升性能
- **RMSNorm**：无需均值中心化的层归一化，计算更快
- **多分辨率 RoPE（MRoPE）**：增强的位置编码，分段为 [8, 12, 12]，以改进空间理解
- **扩展上下文**：支持高达 131,072 个令牌

**视觉语言特性：**
- **视觉编码器**：预训练的视觉编码器，用于鲁棒的视觉理解
- **多模态集成**：视觉和文本信息的无缝集成
- **图像和视频支持**：处理静态图像和视频输入
- **灵活的图像处理**：支持可变分辨率图像和每次对话中的多张图像
